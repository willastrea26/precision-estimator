Supabase Database Schema — 12 Tables
Row-Level Security Policies

All 12 tables have RLS enabled. Every table includes user_id uuid REFERENCES auth.users and timestamps (created_at, updated_at). The RLS policy ensures users can only see and edit their own rows — no cross-user data leakage. The projects table cascades deletes down to work_packages, boq_items, documents, quote_requests, risks, variations, programme_items, activity_log and takeoff_items.

Reserve Strategy — ~107 credits remaining after core build

Build the core with all enhancements (~293 credits), pick 1–2 high-value upgrades (~10–20 credits) only if needed, and keep approximately 85–95 credits in reserve for post-launch refinement. You will discover real-world needs only after using the app for actual tenders.
